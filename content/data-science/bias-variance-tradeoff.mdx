# El Problema

Cuando entrenamos un modelo de machine learning, queremos que funcione bien en datos **que nunca ha visto**. Esto es generalización.

Pero hay una tensión fundamental:

- **Bias alto** → El modelo es demasiado simple, no captura los patrones reales
- **Variance alta** → El modelo es demasiado complejo, memoriza el ruido

## Formalización Matemática

El error de predicción se descompone en:

$$
\text{Error}(x) = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error}
$$

Donde:

- $\text{Bias} = E[\hat{f}(x)] - f(x)$
- $\text{Variance} = E[(\hat{f}(x) - E[\hat{f}(x)])^2]$
- El error irreducible es el ruido inherente en los datos ($\sigma^2$)

## Ejemplo Intuitivo

Imagina disparar flechas a un blanco:

- **Bias bajo, Variance baja** → Flechas agrupadas en el centro ✅
- **Bias alto, Variance baja** → Flechas agrupadas pero lejos del centro
- **Bias bajo, Variance alta** → Flechas dispersas alrededor del centro
- **Bias alto, Variance alta** → Flechas dispersas y lejos del centro

## En la Práctica

```python
from sklearn.model_selection import cross_val_score
from sklearn.tree import DecisionTreeRegressor

# Modelo simple (bias alto, variance baja)
simple_model = DecisionTreeRegressor(max_depth=2)

# Modelo complejo (bias bajo, variance alta)
complex_model = DecisionTreeRegressor(max_depth=50)

# Validación cruzada revela el tradeoff
scores_simple = cross_val_score(simple_model, X, y, cv=5)
scores_complex = cross_val_score(complex_model, X, y, cv=5)
```

## Técnicas para Controlar el Tradeoff

1. **Regularización** → Penaliza complejidad ($L_1$, $L_2$)
2. **Cross-validation** → Estima error de generalización
3. **Ensemble methods** → Bagging reduce variance, Boosting reduce bias
4. **Early stopping** → Detiene antes de overfitting

## Reflexión

El tradeoff bias-variance no es solo una idea de ML. Es una lección sobre **la tensión entre simplicidad y precisión** que aparece en toda la ingeniería.
